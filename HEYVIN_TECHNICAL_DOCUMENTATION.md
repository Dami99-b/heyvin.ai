*A complete handbook tailored for non-technical founders, judges, and investors.*

## Executive Summary & Vision
Heyvin AI was conceived to address a severe, invisible challenge faced by modern students and young professionals: **the collision of spontaneous domestic chores and demanding learning targets**. 

For millions of users residing in multi-generational households, study block interruptions are not merely occasional; they are cultural, frequent, and unpredictable. Heyvin AI acts as a **Sovereignty Shield**—a local-first companion that dynamically maps household friction, schedules protective "study-lock" slots, trains users in boundary assertion, and cushions real-life interruptions with server-free security.

## 🚀 1. Complete Feature Directory

Every capability designed in Heyvin AI solves a direct, real-world friction point:

### A. Today's Sovereignty Hub (Friction Planner)
*   **What it does:** Allows users to log the hour-by-hour friction points they encounter (e.g., parental requests, shared bathroom conflicts, sudden kitchen duties).
*   **Why it was built:** Generic calendar tools assume a static, peaceful desk. Heyvin assumes a dynamic, high-interruption room. Logging when interruptions occur helps users anticipate domestic roadblocks before sitting down to read.

### B. Sovereign Success Checklist (Dynamic Priorities)
*   **What it does:** A three-priorities-only task dashboard emphasizing academic blocks, boundary setups, and career milestones.
*   **Why it was built:** Oversaturated to-do lists trigger anxiety and task paralysis. Grounding user focus in three high-contrast daily sovereignty priorities drives realistic completion.

### C. Calm Zone & Assertive Coping Scripts
*   **What it does:** An off-line meditation tool providing rhythmic breathing guides and polite, culturally compassionate boundary scripts (e.g., "How to respectfully decline an interruption during an active exam").
*   **Why it was built:** Grounding ourselves is useless if we lack the communication frameworks to maintain quiet environments. This tool bridges the gap between inner calm and outward assertion.

### D. Friction Heatmap & Stress Pattern Analytics
*   **What it does:** Generates intuitive color indicators mapping which days of the week and slots of the day experience high, medium, or low household interruption trends.
*   **Why it was built:** Empowering students with hard historical evidence about their most interrupted windows helps them strategically rearrange study hours or shift critical classes to quiet gaps.

### E. Sister Hotline AI Companion
*   **What it does:** A supportive, server-side AI mentor that provides direct advice, boundary negotiation scripts, and emotional coping support.
*   **Why it was built:** When users feel overwhelmed by conflicting chores and career pressures, they require more than a calendar—they need empathetic validation and strategic scripts to navigate parental expectations.

### F. Paystack High-Fidelity Gate
*   **What it does:** Simulates instant premium checkout upgrades for Sovereign Gold features with fully interactive PIN & OTP verification windows.
*   **Why it was built:** Built to demonstrate the business model monetization flow to investors in a simulated, secure checkout workflow without requiring complex configurations.

```

                            │ (Client API Proxy)
                            ▼
┌────────────────────────────────────────────────────────┐
│                      BACKEND                           │
│   • Custom Node.js + Express Server (Port 3000)        │
│   • Secure API Routes (/api/*)                        │
│   • Interactive Google Sign-On Mock Engine             │
└───────────────────────────┬────────────────────────────┘
                            │ (Secure Communication)
                            ▼
┌────────────────────────────────────────────────────────┐
│             AI COOPERATION & DATABASES                 │

│   • Local Device Cryptographic State (LocalStorage)   │
│   • Firebase / Supabase Storage Ready                  │
└────────────────────────────────────────────────────────┘
```

### Stack Components Breakdown:
1.  **Framework:** React 18. Developed using standard modular paths.
2.  **Dev Server:** Vite. Configured to bind on port `3000` to serve index files seamlessly.
3.  **Styles & Font:** Styled with Tailwind utility classes. Configured with a warm sand and rich cocoa-red palette to ensure maximum text visibility under daylight or low light.
4.  **AI Engine:** Uses the modern Google GenAI library. It processes all strategic sister advice, summary audits, and emotional negotiation briefs server-side, protecting the system API key from client visibility.

## 🛠️ 3. Step-by-Step Construction & Critical Decisions

### Phase I: The Core User Validation Flow
*   **Challenge:** Standard Google OAuth popups are easily blocked by sandbox frames or school browsers, completely halting the pitch on judge screens.
*   **Our Decision:** We designed a robust Dual-Sign-On flow. It starts with interactive mock-Google account selection and seamlessly falls back onto a secure local walkthrough container. **Critical Change:** We enforce password creation even with Google authing, ensuring users always establish a local security vault key.

### Phase II: Design Resolution for Smaller Screens (Laptops)
*   **Challenge:** Pitch judges looking at screens on medium laptops often see cut-off continue buttons, cropped text, and faded input outlines due to poor monitor contrast.
*   **Our Decision:** We replaced the fixed, rigid grid ratios with a fluid overflow system. We darkened input outlines from `border-gray-250` (faint) to `border-gray-400` (crisp), and colored text labels deep cocoa (`#3E2E28`).

### Phase III: The Humanization of Data Inputs
*   **Challenge:** Filling long, complex forms ruins first-user onboarding.
*   **Our Decision:** We built interactive, visual choice buttons. For country selectors, we added landscape and national emblem background illustrations backed by frost-glass names to make the choice immersive, fast, and satisfying.

## 🛡️ 5. Privacy, Integrity & Absolute Security

For our user base, privacy is not a passive feature—it is a matter of home safety:

1.  **API Shielding:** The Google Gemini API key is completely isolated inside Node.js environment variables. No client-side browser requests ever touch the LLM endpoint directly, eliminating API request interception.
2.  **Zero Telemetry:** The application is designed strictly without user telemetry tracking, system pinging, or unrequested log recording. 
3.  **LocalStorage Encryption:** User credentials, journal notes, diaries, goals, and household maps are retained strictly inside the user's localized browser storage environment, giving them utter data ownership.
